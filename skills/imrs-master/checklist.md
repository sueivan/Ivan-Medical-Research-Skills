# IMRS Master Quality Checklist

## Classification
- [ ] Research product is explicit
- [ ] Primary and secondary task classes are assigned
- [ ] Readiness state is assigned
- [ ] Consequential ambiguity is resolved

## Routing
- [ ] Every target skill is confirmed operational
- [ ] Route is the minimum sufficient chain
- [ ] Method is chosen before software
- [ ] Unsupported capabilities are disclosed
- [ ] Scaffolds are not treated as operational

## Handoffs
- [ ] Each transition has required inputs
- [ ] Output schema and verification state are specified
- [ ] Stop conditions are explicit
- [ ] Unresolved issues propagate to the next stage

## Quality gates
- [ ] Question gate assessed
- [ ] Vocabulary gate assessed when applicable
- [ ] Search gate assessed when applicable
- [ ] Identity gate assessed when applicable
- [ ] Extraction gate assessed when applicable
- [ ] Corpus gate assessed when applicable
- [ ] Interpretation gate assessed

## Integrity and completion
- [ ] No execution, result, identifier, or count is invented
- [ ] Draft/run/tested/verified states are preserved
- [ ] Completed and unexecuted work are distinguished
- [ ] Access limitations are reported
- [ ] Route, versions, assumptions, deviations, and next action are logged
