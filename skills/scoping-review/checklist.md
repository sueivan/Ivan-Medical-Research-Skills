# Scoping Review Quality Checklist

## Rationale and protocol
- [ ] Scoping review is justified against alternative review types
- [ ] Population/Participants, Concept, and Context are explicit
- [ ] Primary question and subquestions align with charting
- [ ] Evidence-source types and boundaries are operational
- [ ] Protocol status is accurate
- [ ] Amendments record timing, rationale, and impact

## Search and records
- [ ] Sources match the disciplines and evidence types
- [ ] Grey-literature and website needs are considered
- [ ] Every strategy is database specific and reproducible
- [ ] Website searches include path, depth, date, and stopping rule
- [ ] Unrun searches are labeled draft
- [ ] Raw exports, provenance, deduplication rules, and counts are archived

## Selection and linkage
- [ ] Criteria were piloted by reviewers
- [ ] Independent screening decisions are preserved
- [ ] Automation does not make opaque final exclusions
- [ ] Full-text retrieval and specific exclusion reasons are logged
- [ ] Unit of mapping is defined
- [ ] Multiple reports/versions/projects are linked appropriately

## Charting and appraisal
- [ ] Charting form and data dictionary were piloted
- [ ] Fields answer the review questions
- [ ] Iterative changes and back-updates are recorded
- [ ] Reported, inferred, unclear, and missing values are separated
- [ ] Author-stated and reviewer-identified gaps are separated
- [ ] Critical-appraisal decision is justified
- [ ] Appraisal tools are design appropriate if used

## Mapping and reporting
- [ ] Categories and denominators are defined
- [ ] Counts are not interpreted as effectiveness or quality
- [ ] Gap types are specific and evidence based
- [ ] Search restrictions are considered when interpreting gaps
- [ ] Consultation methods and effects are transparent if used
- [ ] PRISMA-ScR counts derive from logs
- [ ] Conclusions remain within the mapping objective
