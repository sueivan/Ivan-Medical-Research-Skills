# PubMed、WoS 与 Scopus 转换

## 原则

转换是概念级重建，不是查找替换。保留原数据库版本，并为每个概念记录新增、删除和语法变化。

## PubMed → Scopus

- MeSH 无直接等价项；提取主题词、入口词和相关自由词，再在 `TITLE-ABS-KEY` 等字段测试。
- PubMed `[tiab]` 可近似映射到题名/摘要字段，但 Scopus 的 KEY 会扩展覆盖，需单独决定。
- PubMed 自动词语映射、短语和截词行为不能假定在 Scopus 相同。
- PubMed 研究类型过滤器需重新验证 Scopus 字段和文献类型，避免高漏检。

## WoS → Scopus

- WoS `TS=` 与 Scopus `TITLE-ABS-KEY` 字段组成并不完全相同。
- WoS `NEAR/x` 需根据是否要求词序转换为 Scopus `W/n` 或 `PRE/n` 并实测。
- 截词、引号、精确匹配和优先级需重建。
- WoS 文献类型、索引和年份限制不可直接复制。

## 转换审计

为每个数据库报告：平台、字段范围、受控词表、邻近、通配符、日期边界、文献类型、结果数、运行日期和验证文献。不同数据库结果数不可用来判断哪一个“更正确”。
