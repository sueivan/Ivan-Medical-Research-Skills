# Contributing

## Contribution standard

Every contribution must improve methodological reliability, reproducibility, or usability.

## Required for every skill

- valid YAML frontmatter
- a precise trigger-based description beginning with `Use when`
- explicit scope and non-scope
- no fabricated data or references
- reproducibility requirements
- common failure modes
- completion criteria
- at least one example prompt
- reviewer-grade quality checks

## Prohibited

- invented PMID, DOI, statistics, findings, software outputs, or journal metrics
- automatic claims that cannot be traced to evidence
- decorative prompts without a reproducible workflow
- replacing expert judgment with generic language generation
- accepting software-generated cluster labels without inspection

## Pull-request expectations

A pull request should explain:

1. what research problem the skill addresses;
2. what failure modes it prevents;
3. how it was tested;
4. what remains uncertain;
5. whether external standards or reporting guidelines are involved.

See `docs/SKILL-STANDARD.md`.
