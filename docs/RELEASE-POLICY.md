# Release Policy

## Release criteria

A stable release requires:

- all included skills pass structural validation;
- no unresolved fabricated-reference risks;
- no known critical methodological errors;
- examples and checklists are present;
- changelog and roadmap are updated.

## Release labels

- `experimental`: usable for testing, not stable
- `beta`: workflow mostly stable, still under validation
- `stable`: suitable for routine use
- `deprecated`: retained only for compatibility
